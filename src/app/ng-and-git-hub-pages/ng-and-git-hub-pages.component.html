5. Web Dev with GitHub Pages and Angular

This tutorial guides you through the process of deploying your first basic web application using Angular and GitHub Pages, via the angular-cli-ghpages package, and includes a summary of considerations, limitations, and inspiration to take when building your own apps with this stack.

This tutorial focuses on the process from zero to deployment rather than on the syntax and tradeoffs of Angular. See the Angular Basics tutorial for an overview of Angular syntax.

5.1. What can we do with GitHub Pages?

GitHub Pages is a 'static' site hosting solution offered by GitHub. Although there are paid tiers, the free tier (for individuals) is a great option for hosting small, personal projects.

Usage of GitHub Pages must firstly abide by the specified Terms of Service and other limitations, most of which are outlined here.

Besides these, there are a few more important points which limit what we are able to host here.

  1. Code must be pushed to GitHub for deployments to work, so you will likely use GitHub to host your project repository.
  2. Only 'static' websites are supported. This means that all HTML/CSS/JS content must be sent to the user upon their browser opening the page. This is in contrast to, a server which can send new HTML/CSS/JS as a user navigates through the application.
  3. Note that the term 'static website' is not a well-defined term, but websites can be thought of on a spectrum from static to dynamic. Strict definition of 'static' say that no API calls beside the initial load can be made (e.g. JavaScript fetching images or data from another URL). For our purposes, we consider 'static' sites as those requiring minimal calls to such additional resources.
  4. As with any website you decide to host (not just static ones, and not only those on GitHub Pages), cybersecurity should be a consideration. For example, you can choose to have your front-end make requests to your own separate back-end server, but it is easy for people with a small amount of programming experience to extract your back-end server's URL once the front-end makes a call to it, which can expose your back-end application to all sorts of attacks.

With all of those considerations in mind, we will now walk through setting up our own Angular application, with support to easily deploy to GitHub pages, and then look at some more examples of what we can do with our applications.

5.2. Installation and Setup

This section guides you through setting up Git and Angular locally, and creating a new repository in your GitHub account. If you already have any of these dependencies installed, you do not need to re-install them.

  1. Install Git locally.
  2. Install Visual Studio Code locally, or alternatively use your preferred IDE.
  3. Follow the Angular Documentation to set up Angular locally. Note that Node is required as a dependency.
  4. Create a GitHub account if you do not already have one.
  5. Once logged in, create a new public repository. Note the name of the repository, as this will be needed later.

5.3. A Minimal App

  1. Using your command line terminal of choice, set your working directory to a desired folder, and run ng new and follow the prompts to create a template app. This command creates a new folder within your working directory which contains your application.
  2. Open the newly created application folder in Visual Studio Code (or your preferred IDE)
  3. Run ng serve in your application folder to locally serve the application, and open the provided link (usually to http://localhost:4200). Congratulations, you have an Angular application!
  4. The default page you see is defined in src/app/app.component.html. Try making some modifications and see how they reflect in the browser. For example, add some text content under the existing text, or try adding a new link pill on the right sidebar. We will leave this for now to focus on the rest of the deployment pipeline, and return later.
  5. Stage your changes in Git with git add . then create a commit with your changes with git commit -m "<YOUR COMMIT MESSAGE HERE>"
  6. Go to your GitHub repository page as created earlier, and follow the prompts to push. Refresh the page, and if successful you will see your code in the GitHub repository.

5.4. angular-cli-ghpages

  1. From the angular-cli-ghpages web page, follow steps 3-5 of the Quick Start guide to install and deploy your app to GitHub pages. Note that you will need the repository name as created earlier.
  2. After the deployment is complete from angular-cli-ghpages, it can take a few minutes for the changes to take place. Under 'Settings' > 'Pages' in your GitHub repository, you can see if your page has been deployed yet, and how long ago it was deployed.

5.5. Examples and Inspiration

Now that we have a way to deploy Angular applications to GitHub Pages, what can we do with our apps? Bearing in mind the limitations and considerations from earlier, here are some ideas for inspiration:

  1. Mathematical engines and logic can be embedded into your JavaScript code to run purely in the user's browser, like this solver for Parks Puzzles. This example is complete static in the sense that no further data is fetched after the initial HTML/CSS/JS payload, neither from a database nor backend. All dynamic content is provided by user inputs and JavaScript.
  2. Applications can still be data-driven as long as data is embedded within the application, or is sourced from a publicly accessible location, or is provided by the user. This trajectory plotting application, PathViewer begins with an embedded dataset upon loading, but can fetch data from a simple database (the same GitHub repository in this case), and is able to accept user inputs in the appropriate format. As all data in this case is stored in GitHub or provided by users, there is no back-end server that we need to take care of securing or maintaining.
  3. Animation of page content on scroll can be done using CSS and the IntersectionObserver API available via JavaScript in all common browsers. This tutorial from Beyond Fireship demonstrates how to do this (with a plain JavaScript project).
